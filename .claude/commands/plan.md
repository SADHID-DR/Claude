description: Break work into small verifiable tasks with acceptance criteria and dependency ordering
---

Invoke the agent-skills:planning-and-task-breakdown skill.

Break the current objective into a structured task list:

1. **Read-only analysis** — examine existing specs and relevant codebase sections without making changes
2. **Map dependencies** — identify how components relate and depend on each other
3. **Vertical slicing** — organize work so each task delivers complete end-to-end functionality
4. **Write acceptance criteria** — define measurable conditions to verify each task
5. **Add phase checkpoints** — insert validation points between major work phases
6. **Present for approval** — get human sign-off before execution begins

**Output:**
- `tasks/plan.md` — comprehensive breakdown with dependencies and acceptance criteria
- `tasks/todo.md` — actionable task list for tracking progress
