---
description: Write a structured specification before writing code
---

Invoke the agent-skills:spec-driven-development skill.

Before writing any code, gather requirements by asking clarifying questions:

1. **Objective & Users** — What are you building, and who will use it?
2. **Core Features** — What are the essential capabilities and acceptance criteria?
3. **Tech Stack** — What technologies are preferred or required?
4. **Constraints** — Performance requirements, integrations, or limitations?
5. **Boundaries**:
   - What should always be done a certain way?
   - What requires discussion before implementation?
   - What should never be done?

Generate a comprehensive **SPEC.md** at the project root covering:
- Objective
- Commands
- Project Structure
- Code Style
- Testing Strategy
- Boundaries

Confirm with the user before writing any code.
